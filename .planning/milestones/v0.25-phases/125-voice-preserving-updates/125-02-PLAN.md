    - Test: checkHeadingLevels returns { passed: true } when suggestion heading levels match original (e.g., both use == and ===)
    - Test: checkHeadingLevels returns { passed: false, mismatches } when suggestion introduces different heading levels (e.g., original has == but suggestion uses ###)
    - Test: checkFormattingNovelty returns advisory list of novel formatting elements found (e.g., "code blocks added" when original has no code blocks) — advisory only, not blocking
    - Test: checkSectionLength returns advisory note "Consider splitting this section" when suggestion exceeds 150% of original length

  <action>
1. Add to wiki-voice-validator.ts:

   `checkTemplatePreservation(originalText: string, suggestionText: string)`:
   - Extract all `{{TemplateName` patterns from original (just the template name, not params)
   - Check each template name appears in suggestion
   - Return `{ passed: boolean; missingTemplates: string[] }`

   `checkHeadingLevels(originalText: string, suggestionText: string)`:
   - Extract heading patterns from original: MediaWiki style `^={2,6}[^=]+=+$` and/or markdown `^#{1,6}\s`
   - Extract same from suggestion
   - Compare: original heading levels should appear in suggestion at same levels
   - Return `{ passed: boolean; mismatches: string[] }`

   `checkFormattingNovelty(originalText: string, suggestionText: string)`:
   - Check for novel formatting elements in suggestion not in original:
     - Code blocks (``` or <code>/<pre>) when original has none
     - Tables (| or <table>) when original has none
     - Bold (**) / italic (*) patterns when original has none
   - Return `{ novelElements: string[] }` — ADVISORY only, per CONTEXT.md these are encouraged
   - Log the advisory items at debug level

   `checkSectionLength(originalText: string, suggestionText: string)`:
   - Compare character counts
   - If suggestion > 150% of original length, return advisory note
   - Return `{ advisory: string | null }`

2. Update `VoicePreservedSuggestion` type to add:
   ```typescript
   templateCheckPassed: boolean;
   headingCheckPassed: boolean;
   formattingAdvisory: string[];
   sectionLengthAdvisory: string | null;
   ```

3. Wire checks into `generateWithVoicePreservation`:
   - After generating suggestion (and before voice validation):
     a. Run `checkTemplatePreservation`. If failed, regenerate ONCE with explicit feedback about missing templates. If second attempt also fails, DROP the suggestion entirely (return null/empty).
     b. Run `checkHeadingLevels`. Flag mismatches in the result but don't block.
     c. Run `checkFormattingNovelty`. Store advisory in result.
     d. Run `checkSectionLength`. Store advisory in result.
   - Template check failure on second attempt should cause the function to return a result with `suggestion: ""` and `templateCheckPassed: false` to signal the caller to skip this suggestion.

4. Add all tests per the behavior block.
