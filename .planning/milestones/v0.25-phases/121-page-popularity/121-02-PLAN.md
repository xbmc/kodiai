    **Linkshere fetcher — wiki-linkshere-fetcher.ts:**
    Follow the MediaWiki API fetching pattern from `wiki-sync.ts`. Export `fetchAllLinkshereCounts()`:

    ```typescript
    export async function fetchAllLinkshereCounts(opts: {
      baseUrl: string;          // e.g., "https://kodi.wiki"
      pageIds: number[];        // All wiki page_ids to query
      fetchFn?: typeof globalThis.fetch;
      logger: Logger;
    }): Promise<Map<number, number>>
    ```

    Implementation:
    - Batch pageIds into groups of `LINKSHERE_BATCH_SIZE` (50) per request
    - For each batch, call MediaWiki API: `action=query&prop=linkshere&pageids={batch.join("|")}&lhprop=pageid&lhlimit=500&lhnamespace=0&format=json`
    - Handle pagination: check `response.continue?.lhcontinue` and loop with `lhcontinue` parameter until complete
    - Cap per-page accumulation at `LINKSHERE_MAX_PER_PAGE` (5000) to prevent runaway pagination
    - Delay `LINKSHERE_RATE_LIMIT_MS` (500ms) between batch requests
    - Return Map<pageId, linkCount>
    - Fail-open: if a batch fails, log warning and continue with remaining batches (do not abort entire fetch)
    - Import constants from `wiki-popularity-config.ts`

    **Popularity scorer — wiki-popularity-scorer.ts:**
    Follow the `createWikiStalenessDetector` scheduler pattern exactly. Export:

    ```typescript
    export type WikiPopularityScorerOptions = {
      sql: Sql;
      logger: Logger;
      wikiPageStore: WikiPageStore;
      popularityStore: ReturnType<typeof createWikiPopularityStore>;
      wikiBaseUrl: string;           // e.g., "https://kodi.wiki"
      fetchFn?: typeof globalThis.fetch;
      intervalMs?: number;           // default: 7 days
      startupDelayMs?: number;       // default: 5 minutes
    };

    export type WikiPopularityScoringResult = {
      pagesScored: number;
      citationsAggregated: number;
      citationsCleaned: number;
      durationMs: number;
      skipped: boolean;
      skipReason?: string;
    };

    export function createWikiPopularityScorer(
      opts: WikiPopularityScorerOptions,
    ): { start(): void; stop(): void; runNow(): Promise<WikiPopularityScoringResult> };
    ```

    The `runScoring()` internal function:
    1. Get all wiki pages from `wikiPageStore` (need page_id, page_title, last_modified) — use the store's existing query methods to get distinct pages
    2. If no pages, return early with `skipped: true`
    3. Fetch inbound link counts via `fetchAllLinkshereCounts()` for all page_ids
    4. Get citation counts from `popularityStore.getCitationCounts(CITATION_WINDOW_DAYS)`
    5. Clean up old citations via `popularityStore.cleanupOldCitations(CITATION_WINDOW_DAYS)`
    6. Compute normalization bounds: min/max of inbound_links across all pages, min/max of citation_count across all pages
    7. For each page, compute `daysSinceEdit = (Date.now() - new Date(page.last_modified).getTime()) / (1000 * 60 * 60 * 24)`
    8. Call `computeCompositeScore()` from config module for each page
    9. Upsert all results via `popularityStore.upsertPopularity()`
    10. Log summary: pages scored, citation window, cleanup count

    Scheduler pattern (from wiki-staleness-detector.ts):
    - `let intervalHandle`, `startupHandle`, `running` guard
    - `start()`: setTimeout for startup delay, then setInterval for recurring
    - `stop()`: clearTimeout + clearInterval
    - `runNow()`: calls doScore() with running guard
    - Default interval: 7 * 24 * 60 * 60 * 1000 (weekly)
    - Default startup delay: 5 * 60 * 1000 (5 minutes)

    **Getting distinct pages:** The wiki_pages table has multiple rows per page_id (one per chunk). Query distinct page_ids with their page_title and last_modified. Use a SQL query like: `SELECT DISTINCT ON (page_id) page_id, title, last_modified FROM wiki_pages ORDER BY page_id, chunk_index ASC` — or use the wiki store's existing methods if they support this. Check what WikiPageStore exposes (it may have a `listPages` or similar). If not, add a direct SQL query in the scorer.
