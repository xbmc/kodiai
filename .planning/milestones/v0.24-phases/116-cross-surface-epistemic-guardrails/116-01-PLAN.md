## Objective

Propagate epistemic guardrails from the PR review surface to mention responses and Slack assistant, using a single shared source of truth. All surfaces refuse to assert external facts unless grounded in visible context or retrieved enrichment data.

## must_haves

- [ ] `buildEpistemicBoundarySection()` uses surface-neutral language (no review-specific wording)
- [ ] Mention prompt (PR mentions) includes the shared epistemic section with diff-grounding rules
- [ ] Mention prompt (issue mentions) includes the shared epistemic section with context-visible tier adapted for issue body/thread
- [ ] Slack assistant prompt includes the shared epistemic section with appropriate adaptation
- [ ] Existing PR review behavior is unchanged (same epistemic rules, just neutral language)
- [ ] Tests verify epistemic guardrails appear in each surface's prompt output

## Context

Phase 115 added `buildEpistemicBoundarySection()` in `src/execution/review-prompt.ts:238` with a 3-tier epistemic system (diff-visible, system-enrichment, external-knowledge). Currently it uses review-specific language like "Every assertion in this review." This phase generalizes it and integrates it into mention and Slack prompts.

Key files:
- `src/execution/review-prompt.ts` — has `buildEpistemicBoundarySection()` and `buildToneGuidelinesSection()`
- `src/execution/mention-prompt.ts` — `buildMentionPrompt()` with existing "Factual Accuracy — CRITICAL" section (lines 288-303)
- `src/slack/assistant-handler.ts` — `buildSlackAssistantPrompt()` with inline "Never hedge" rule (line 128)
