  </behavior>
  <action>
    Create `src/lib/guardrail/pipeline.ts` with `runGuardrailPipeline<TInput, TOutput>(opts)`:

    1. Extract claims via `adapter.extractClaims(output)`
    2. Build grounding context via `adapter.buildGroundingContext(input)`
    3. Determine effective strictness: check `config.overrides?.[adapter.surface]?.strictness` first, fall back to `config.strictness`
    4. Classify each claim via `classifyClaimAgainstContext(claim, context, strictness)`. Wrap in try/catch -- on error, mark classifierError=true and fail-open (treat all claims as grounded)
    5. Collect ambiguous claims (confidence < 0.6) for potential LLM fallback (placeholder: accept an optional `llmClassifier` callback, if not provided treat ambiguous as grounded -- LLM fallback comes in plan 02)
    6. Filter: keep claims with label "diff-grounded" or "inferential", remove "external-knowledge"
    7. Check minimum content: if kept claims joined text has fewer words than `adapter.minContentThreshold`, suppress entirely (output=null)
    8. Reconstruct output via `adapter.reconstructOutput(output, keptClaims)`
    9. Build audit record with aggregate counts + removed claim details
    10. If auditStore provided, call `auditStore.logRun(record)` (fire-and-forget)
    11. Return `GuardrailResult<TOutput>`

    Pipeline must be fully synchronous-compatible for the classification step (async only for LLM fallback and audit store).

    Tests use a simple mock adapter that operates on string arrays or plain text. No need for real surface adapters yet.
