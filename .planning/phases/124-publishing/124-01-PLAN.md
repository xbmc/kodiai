Create `src/knowledge/wiki-publisher.ts` with a `createWikiPublisher` factory function.

**Imports:**
```typescript
import type { Octokit } from "@octokit/rest";
import type { WikiPublisherOptions, PublishResult, PublishRunOptions, PageSuggestionGroup, PagePostResult } from "./wiki-publisher-types.ts";
import type { UpdateSuggestion } from "./wiki-update-types.ts";
```

**Factory: `createWikiPublisher(options: WikiPublisherOptions)`**

Returns object with single method `publish(runOptions: PublishRunOptions): Promise<PublishResult>`.

Apply defaults: `owner = "xbmc"`, `repo = "wiki"`, `prOwner = "xbmc"`, `prRepo = "xbmc"`, `commentDelayMs = 3000`.

**`publish()` flow:**

1. **Pre-flight check (PUB-04):**
   - Call `githubApp.getRepoInstallationContext(owner, repo)`
   - If returns null: log error "GitHub App not installed on {owner}/{repo}. Install the app at https://github.com/apps/{appSlug}/installations" and return early with empty result (no throw — graceful failure)
   - Get `installationId` from result, call `githubApp.getInstallationOctokit(installationId)` to get authenticated Octokit

2. **Fetch unpublished suggestions:**
   ```sql
   SELECT id, page_id, page_title, section_heading, suggestion, why_summary,
          citing_prs, voice_mismatch_warning
   FROM wiki_update_suggestions
   WHERE published_at IS NULL
     AND grounding_status = 'grounded'
   ORDER BY page_id, section_heading NULLS FIRST
   ```
   - If `pageIds` provided: add `AND page_id = ANY(${pageIds})`
   - If `groundedOnly`: add `AND voice_mismatch_warning = false`
   - If no suggestions found: log info and return early with zero counts

3. **Group by page:**
   Group suggestions into `PageSuggestionGroup[]` — all suggestions for a page go in one comment.

4. **Dry-run branch:**
   If `dryRun`: format all page comments and the summary table to markdown string, log to console, return result with `issueNumber: null`. Skip steps 5-8.

5. **Create tracking issue (PUB-01):**
   ```typescript
   const today = new Date().toISOString().slice(0, 10);
   const issue = await octokit.rest.issues.create({
     owner, repo,
     title: `Wiki Update Suggestions — ${today}`,
     body: "Posting update suggestions... (will be updated with summary table)",
     labels: ["wiki-update", "bot-generated"],
   });
   ```
   Store `issue.data.number` and `issue.data.html_url`.

6. **Post per-page comments (PUB-02 + PUB-03):**
   For each `PageSuggestionGroup`:
   - Format comment body using `formatPageComment(group, prOwner, prRepo)` (helper)
   - Call `postCommentWithRetry(octokit, owner, repo, issueNumber, commentBody)` (helper with rate limiting)
   - On success: store `PagePostResult` with `commentId`
   - On failure after retries: store as skipped, continue to next page
   - Wait `commentDelayMs` between each comment (even on failure)

7. **Mark published in DB:**
   For each successful page post, UPDATE all that page's suggestions:
   ```sql
   UPDATE wiki_update_suggestions
   SET published_at = NOW(), published_issue_number = ${issueNumber}
   WHERE page_id = ${pageId} AND published_at IS NULL AND grounding_status = 'grounded'
   ```

8. **Update issue body with summary table:**
   Build summary table with columns: #, Page, Wiki Link, Sections, PRs Cited, Voice Warnings, Comment
   - Posted pages get anchor links: `[View](#issuecomment-${commentId})`
   - Skipped pages get "skipped" with error reason
   - Call `octokit.rest.issues.update({ owner, repo, issue_number, body: summaryBody })`

**Helper: `formatPageComment(group, prOwner, prRepo): string`**
```markdown
## {pageTitle}

**Wiki page:** [View on wiki](https://kodi.wiki/view/{encodeURIComponent(pageTitle)})

---

### {sectionHeading || "Introduction"}

{suggestion}

**Why:** {whySummary}
**PRs:** [#{prNumber}](https://github.com/{prOwner}/{prRepo}/pull/{prNumber}) ({prTitle}), ...

> ⚠️ **Voice mismatch** — review tone and style carefully before applying
(only if voiceMismatchWarning is true)

---
(repeat for each section)
```

**Helper: `formatSummaryTable(date, pageResults, skippedPages): string`**
Returns the full issue body markdown with the summary table.

**Helper: `postCommentWithRetry(octokit, owner, repo, issueNumber, body, maxRetries = 3): Promise<{ commentId: number } | null>`**
- Post comment via `octokit.rest.issues.createComment`
- On 403: check `Retry-After` header, wait that many seconds (or 60s default), retry
- Exponential backoff: 60s, 120s, 240s
- After `maxRetries` failures: return null
- On other errors: return null immediately (no retry)

**Helper: `delay(ms): Promise<void>`**
Simple setTimeout wrapper.
