    - selectExemplarSections groups chunks by sectionHeading and selects distinct sections

  <action>
    1. Create `src/knowledge/wiki-voice-types.ts` with types:
       - `PageStyleDescription`: `{ pageTitle: string; styleText: string; formattingElements: string[]; mediaWikiMarkup: string[]; tokenCount: number }`
       - `VoiceValidationResult`: `{ passed: boolean; scores: { toneMatch: number; perspectiveMatch: number; structureMatch: number; terminologyMatch: number; formattingMatch: number; markupPreservation: number }; overallScore: number; feedback: string | null }`
       - `VoiceAnalyzerOptions`: `{ taskRouter: TaskRouter; costTracker?: CostTracker; logger: Logger; repo?: string }`
       - `StyleExemplar`: `{ sectionHeading: string; chunkText: string; chunkIndex: number }`

    2. Create `src/knowledge/wiki-voice-analyzer.ts` with:
       - `selectExemplarSections(chunks: WikiPageRecord[], targetCount?: number): StyleExemplar[]`
         - Group chunks by sectionHeading (null heading = intro section)
         - Filter out sections where total chunkText length < 50 chars
         - If remaining sections <= targetCount (default 3), return all
         - Otherwise, select from spread positions: pick indices at Math.floor(i * sectionCount / targetCount) for i in 0..targetCount-1
         - Prefer sections containing formatting diversity (check for: bullet lists `*` or `-`, numbered lists `1.`, links `[[` or `[http`, code blocks, `{{` templates)
         - Return as StyleExemplar array

       - `extractPageStyle(pageChunks: WikiPageRecord[], opts: VoiceAnalyzerOptions): Promise<PageStyleDescription>`
         - Select representative content: use first ~3000 tokens of page content (sum chunk tokenCount, stop when budget exceeded)
         - Build style extraction prompt that asks the LLM to analyze: tone, perspective, sentence structure, terminology, formatting conventions, and MediaWiki-specific markup
         - Call `generateWithFallback` with taskType `voice.extract`, the resolved model from `opts.taskRouter.resolve("voice.extract")`
         - Parse response text into PageStyleDescription (styleText = full response text, extract formatting elements and MediaWiki markup via simple regex: `{{...}}` patterns, heading patterns, list patterns)
         - Return the PageStyleDescription

    3. Write tests in `src/knowledge/wiki-voice-analyzer.test.ts`:
       - Test selectExemplarSections with the behaviors listed above
       - Use mock WikiPageRecord arrays with known sectionHeadings, chunkText lengths, and chunkIndexes
