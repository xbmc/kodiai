  <behavior>
    - parseVoiceValidation extracts 6 dimension scores from LLM response text (TONE_MATCH, PERSPECTIVE_MATCH, STRUCTURE_MATCH, TERMINOLOGY_MATCH, FORMATTING_MATCH, MARKUP_PRESERVATION)
    - parseVoiceValidation returns passed=true when average score >= 3.5
    - parseVoiceValidation returns passed=false when average score < 3.5
    - parseVoiceValidation extracts FEEDBACK text after "FEEDBACK:" line
    - parseVoiceValidation handles malformed LLM responses gracefully (returns passed=false with feedback explaining parse failure)
    - validateVoiceMatch calls generateWithFallback with voice.validate task type
    - generateWithVoicePreservation retries once on validation failure with feedback injected
    - generateWithVoicePreservation sets voiceMismatchWarning=true if retry also fails
    - generateWithVoicePreservation returns suggestion unchanged if validation passes first time
  </behavior>
  <action>
    1. Create `src/knowledge/wiki-voice-validator.ts` with:

       - `VOICE_MATCH_THRESHOLD = 3.5` — exported constant for the pass/fail cutoff

       - `parseVoiceValidation(text: string): VoiceValidationResult` (exported for testing)
         - Parse dimension scores via regex: `/TONE_MATCH:\s*(\d+)/`, etc. for all 6 dimensions
         - Calculate overallScore as average of 6 dimensions
         - passed = overallScore >= VOICE_MATCH_THRESHOLD
         - Extract feedback: text after "FEEDBACK:" line (null if not present or if passed)
         - On parse failure (missing scores): return { passed: false, scores: all zeros, overallScore: 0, feedback: "Voice validation response could not be parsed" }

       - `validateVoiceMatch(opts: { originalSection: string; generatedSuggestion: string; styleDescription: PageStyleDescription; } & VoiceAnalyzerOptions): Promise<VoiceValidationResult>`
         - Build validation prompt (as described in RESEARCH.md) comparing original vs generated
         - Include style description text and ask for 6 dimension scores (1-5 each)
         - Call `generateWithFallback` with taskType `voice.validate`
         - Parse response with `parseVoiceValidation`
         - Log scores at debug level: `{ pageTitle, overallScore, passed, dimensions: scores }`
         - Return VoiceValidationResult

       - `VoicePreservedSuggestion` type: `{ suggestion: string; voiceMismatchWarning: boolean; validationResult: VoiceValidationResult }`

       - `generateWithVoicePreservation(opts: { generateFn: () => Promise<string>; originalSection: string; styleDescription: PageStyleDescription; buildPromptWithFeedback: (feedback: string) => Promise<string>; } & VoiceAnalyzerOptions): Promise<VoicePreservedSuggestion>`
         - Call opts.generateFn() to get initial suggestion
         - Call validateVoiceMatch with the suggestion
         - If passed: return { suggestion, voiceMismatchWarning: false, validationResult }
         - If failed: log warning, call opts.buildPromptWithFeedback(validation.feedback) to get regenerated suggestion
         - Call validateVoiceMatch again on the regenerated suggestion
         - If passed: return { suggestion: regenerated, voiceMismatchWarning: false, validationResult }
         - If still failed: log warning with "voice mismatch" tag, return { suggestion: regenerated, voiceMismatchWarning: true, validationResult }

    2. Write tests in `src/knowledge/wiki-voice-validator.test.ts`:
       - Test parseVoiceValidation with well-formed LLM output (all scores present, PASS case)

         - Case 3: Both fail — returns with voiceMismatchWarning=true
