Add explicit instructions to the write-mode system prompt section:

```
IMPORTANT: Branch and push policy:
- You are operating in a fork of the target repository. Your git remote "origin" points to the fork, NOT the target repo.
- Do NOT attempt to create branches directly in the target repository.
- Do NOT attempt to push directly to the target repository.
- All your git operations (checkout -b, commit, push) go through the fork automatically.
- When creating pull requests, they will be cross-fork PRs from the fork to the upstream repository -- this is handled by the system, not by you.
- If you need to share a patch without a PR, the system will create a gist automatically based on the nature of the change.
```

  <name>Task 2: Create legacy branch cleanup script</name>
  <files>scripts/cleanup-legacy-branches.ts</files>
  <action>
Create `scripts/cleanup-legacy-branches.ts` -- a one-time script to delete legacy `kodiai/write-*` and `kodiai/slack/*` branches from target repos that were created by the old direct-push approach.

The script should:

1. Accept command-line args: `--owner <owner>` (required), `--repo <repo>` (optional, defaults to all repos for the installation), `--dry-run` (default true, must pass `--no-dry-run` to actually delete)

2. Use the GitHub App auth (import from `src/auth/github-app.ts` and `src/config.ts`) to get an installation token for the owner's repos.

3. For each repo (or the specified repo):
   - List all refs matching `kodiai/` prefix using `octokit.rest.git.listMatchingRefs({ owner, repo, ref: "heads/kodiai/" })`
   - Filter to only `kodiai/write-*` and `kodiai/slack/*` patterns
   - In dry-run mode: print the branches that would be deleted
   - In delete mode: call `octokit.rest.git.deleteRef({ owner, repo, ref: ref.ref.replace("refs/", "") })` for each

4. Print summary: total branches found, deleted (or would be deleted in dry-run)

5. Run with: `bun scripts/cleanup-legacy-branches.ts --owner xbmc --dry-run`

Use existing patterns from other scripts in the `scripts/` directory for argument parsing and GitHub App initialization.
