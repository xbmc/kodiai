**1. src/jobs/fork-manager.ts (4 errors) -- type-narrow split result:**

In `waitForForkReady()` at line 47 and in `ensureFork()` at line 75, `response.data.full_name.split("/")` returns `string[]` but destructuring gives `string|undefined` per TypeScript strict mode.

Fix both locations by adding a type assertion after split:
```typescript
const parts = response.data.full_name.split("/") as [string, string];
const [forkOwner, forkRepo] = parts;
```

This is safe because GitHub `full_name` is always `owner/repo` format. Apply this fix at both line 47 (in `waitForForkReady`) and line 75 (in `ensureFork`).

**2. src/jobs/gist-publisher.ts (1 error) -- non-null assertion on gist id:**

At line 49, `response.data.id` is typed as `string|undefined` by Octokit. The `html_url` on line 48 already uses `!` assertion. Add the same to `id`:
```typescript
const id = response.data.id!;
```

This is safe because a successful `gists.create` always returns an id.
