Wire fork-based write mode into the two write-mode consumer flows (mention handler and Slack write-runner). Change workspace creation to clone from the fork. Add output routing (gist vs PR). Add cross-fork PR creation. Add fallback to gist on fork/PR failure.

Purpose: This is the core behavioral change -- write operations stop creating branches in target repos and instead use the bot-owned fork. Gist output becomes first-class for simple changes.

Output: Modified workspace, mention handler, write-runner, index.ts, and types.

  <name>Task 1: Fork-aware workspace creation and output routing helper</name>
  <files>src/jobs/workspace.ts, src/jobs/types.ts</files>
  <action>
1. In `src/jobs/types.ts`, add an optional `forkContext` to `CloneOptions`:
```typescript
export interface CloneOptions {
  owner: string;
  repo: string;
  ref: string;
  depth?: number;
  /** If provided, clone from the fork instead of the target repo. */
  forkContext?: {
    forkOwner: string;
    forkRepo: string;
    botPat: string;
  };
}
```

2. In `src/jobs/workspace.ts`, modify `createWorkspaceManager.create()`:
   - If `options.forkContext` is provided, build the clone URL using `options.forkContext.botPat` instead of the installation token: `https://x-access-token:${botPat}@github.com/${forkOwner}/${forkRepo}.git`
   - After cloning from fork, add an `upstream` remote pointing at the original repo (using installation token) for reference
   - The `origin` remote naturally points at the fork, so all existing `git push origin ...` calls in `createBranchCommitAndPush`, `commitAndPushToRemoteRef`, and `pushHeadToRemoteRef` will push to the fork without modification

3. Add a guard function exported from `workspace.ts`:
```typescript
/** Verify that origin points to the expected fork, not the upstream repo. Throws if mismatch. */
export async function assertOriginIsFork(dir: string, expectedForkOwner: string): Promise<void>
```
   - Read `git remote get-url origin` and verify it contains `github.com/${expectedForkOwner}/`
   - If not, throw an Error with descriptive message about the push guard preventing direct pushes to target repos

4. Add the output routing helper as a standalone exported function in `workspace.ts` (used by both mention.ts and write-runner.ts):
```typescript
export function shouldUseGist(intent: { keyword?: string }, changedFiles: string[]): boolean
```
   - If `intent.keyword === "patch"` return true
   - If `intent.keyword === "pr"` return false
   - If `changedFiles.length === 1` return true (single file = gist)
   - If `changedFiles.length > 3` return false (many files = PR)
   - For 2-3 files: check if all in same directory, if so return true, else false
