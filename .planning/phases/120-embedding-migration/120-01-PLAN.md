**In `src/knowledge/embeddings.ts`:**

Add a new `createContextualizedEmbeddingProvider` function below the existing `createEmbeddingProvider`. This provider:
- Accepts the same opts shape: `{ apiKey, model, dimensions, logger }`
- Creates a `VoyageAIClient` instance
- Returns an `EmbeddingProvider` where `generate()` calls `client.contextualizedEmbed()` instead of `client.embed()`
- For `contextualizedEmbed`, wraps the single text as `inputs: [[text]]` (one document with one chunk). This works for both document and query embedding of single strings.
- Passes `outputDimension: dimensions` and `inputType` through
- Response structure: `response.data?.[0]?.data?.[0]?.embedding` (document index 0, chunk index 0)
- Same fail-open semantics: catch VoyageAIError and generic errors, log warnings, return null
- Same timeout/retry config: `{ timeoutInSeconds: 10, maxRetries: 2 }`
- Export the function

Also add a **batch-capable helper** `contextualizedEmbedChunks` that is exported for the backfill script to use:
```typescript
export async function contextualizedEmbedChunks(opts: {
  client: VoyageAIClient;
  chunks: string[];
  model: string;
  dimensions: number;
  logger: Logger;
}): Promise<Map<number, Float32Array>>
```
- Sends all chunks as a single document: `inputs: [chunks]` (one inner array = one document with shared context)
- `inputType: "document"`, `outputDimension: dimensions`
- Returns a Map from chunk index to Float32Array embedding
- Iterates `response.data?.[0]?.data` and maps `item.index` to `new Float32Array(item.embedding)`
- Fail-open: returns empty Map on error
- Also export the `VoyageAIClient` re-export from voyageai for use by the backfill script

Parameterize the `createWikiPageStore` factory to accept an optional `embeddingModel` string:

- In `replacePageChunks` (line 131): same change
- This allows the caller to set the model name used when storing chunks

Per user decision (EMBED-03): wiki-store accepts embedding model as a parameter instead of hardcoding.
