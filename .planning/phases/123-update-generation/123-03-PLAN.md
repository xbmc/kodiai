Create `scripts/generate-wiki-updates.ts` following the pattern from `scripts/backfill-wiki.ts`:

**Header comment:**
```typescript
/**
 * CLI script for generating wiki update suggestions from PR evidence.
 *
 * Processes top N stale wiki pages (by popularity score) and generates
 * section-level rewrite suggestions grounded in actual code diffs.
 *
 * Usage:
 *   bun scripts/generate-wiki-updates.ts                    # Process top 20 pages
 *   bun scripts/generate-wiki-updates.ts --top-n 5          # Process top 5 pages
 *   bun scripts/generate-wiki-updates.ts --page-ids 123,456 # Specific pages
 *   bun scripts/generate-wiki-updates.ts --dry-run           # Generate but don't store
 *   bun scripts/generate-wiki-updates.ts --rate-limit 500    # 500ms between LLM calls
 *
 * Environment variables required:
 *   DATABASE_URL          - PostgreSQL connection string
 *   ANTHROPIC_API_KEY     - For LLM generation (or other provider keys)
 */
```

**Arguments via `parseArgs`:**
- `--top-n` (string, default "20") — Number of top pages to process
- `--page-ids` (string, optional) — Comma-separated page IDs to target (overrides top-n)
- `--dry-run` (boolean, default false) — Skip DB writes
- `--rate-limit` (string, default "300") — Milliseconds between LLM calls
- `--help` (boolean) — Show usage

**Script flow:**

1. Parse arguments, show help if --help
2. Create pino logger at LOG_LEVEL (default "info")
3. Create DB client via `createDbClient`
4. Run migrations via `runMigrations`
5. Create wiki page store via `createWikiPageStore` — needs embedding provider, use `createNoOpEmbeddingProvider()` since we're not doing embeddings in this script
6. Create task router via `createTaskRouter`
7. Create cost tracker via `createCostTracker`
8. Parse page IDs if provided: `values["page-ids"]?.split(",").map(Number).filter(n => !isNaN(n))`
9. Create update generator via `createUpdateGenerator({...})`
10. Call `generator.run({ topN, dryRun, pageIds })`
11. Log summary result:
    ```
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    Wiki Update Generation Complete
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    Pages processed:        {n}
    Sections processed:     {n}
    Suggestions generated:  {n}
    Suggestions dropped:    {n} (ungrounded)
    Voice mismatches:       {n}
    Duration:               {n}s
    Mode:                   {dry-run | live}
    ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    ```
12. Close DB connection via `sql.end()`
13. `process.exit(0)`

**Error handling:**
- Wrap main flow in try/catch
- On error: log at error level, close DB, `process.exit(1)`
