**Module design:**

```typescript
import type { FindingSeverity } from "../knowledge/types.ts";
import type { FindingClaimClassification } from "./claim-classifier.ts";

/** Input: a finding with severity and optional claim classification */
export type DemotableFinding = {
  severity: FindingSeverity;
  title: string;
  commentId: number;
  claimClassification?: FindingClaimClassification;
  [key: string]: unknown; // pass-through for all other fields
};

/** Output: finding with severity potentially overwritten and originalSeverity added */
export type DemotedFinding<T extends DemotableFinding> = T & {
  originalSeverity?: FindingSeverity;
  severityDemoted?: boolean;
  demotionReason?: string;
};

export function demoteExternalClaimSeverities<T extends DemotableFinding>(
  findings: T[],
  logger?: { info: (obj: Record<string, unknown>, msg: string) => void },
): DemotedFinding<T>[];
```

**Logic:**
1. For each finding, check `finding.claimClassification?.summaryLabel`
2. If summaryLabel is NOT `"primarily-external"`, return finding unchanged (spread with no extra fields)
3. If summaryLabel IS `"primarily-external"` AND severity is `"critical"` or `"major"`:
   - Set `originalSeverity` = current severity
   - Set `severity` = `"medium"`
   - Set `severityDemoted` = true
   - Set `demotionReason` = build string from claim evidence (collect evidence strings from claims with label "external-knowledge")
   - If logger provided, log at info level: `{ findingTitle: finding.title, originalSeverity, newSeverity: "medium", reason: demotionReason, summaryLabel: "primarily-external" }`, message "Severity demoted: external knowledge claim"
4. If summaryLabel IS `"primarily-external"` but severity is already `"medium"` or `"minor"`, return unchanged
5. If claimClassification is undefined/null, return unchanged (fail-open)

Use immutable spread pattern consistent with classifyClaims: `return { ...finding, severity: "medium", originalSeverity: finding.severity, ... }`

const demotedFindings = demoteExternalClaimSeverities(
