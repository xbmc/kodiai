1. Create `src/knowledge/review-comment-catchup.ts` with the `catchUpReviewComments` function.

2. The function flow:
   a. Get sync state via `store.getSyncState(repo)`. If `!syncState?.backfillComplete`, log info and return early with zeroed result.
   b. Use `syncState.lastSyncedAt` as the `since` parameter. If null, use 24 hours ago as a safe default.
   c. Paginate through `octokit.rest.pulls.listReviewCommentsForRepo({ since, sort: "updated", direction: "asc", per_page: 100 })` wrapped in `withRetry`.
   d. For each page: group into threads using `groupCommentsIntoThreads` (import from backfill).
   e. For each thread, wrap in try/catch (per-thread isolation):
      - For each comment in thread: call `store.getByGithubId(repo, commentGithubId)` to check if it exists
      - If comment does NOT exist -> it's new, proceed with chunk + embed + writeChunks
      - If comment EXISTS and `github_updated_at` from API > stored `githubUpdatedAt` -> it's edited, chunk + embed + `store.updateChunks`
      - If comment EXISTS and dates match -> skip (already stored, no change)
   f. After all pages: update sync state with the latest comment's `updated_at` as new `lastSyncedAt`. Keep `backfillComplete: true`.
   g. Return `CatchUpSyncResult` with counts.

3. Import `withRetry` and `groupCommentsIntoThreads` from `review-comment-backfill.ts`. Also import the `embedChunks` helper -- if it's not exported, export it from backfill (it's a pure helper, safe to export). Import `chunkReviewThread` from `review-comment-chunker.ts`.

4. Use structured pino logs for progress: log at start (repo, since date), per-page (page number, new/updated/skipped counts), and at end (totals, duration).

5. For the dedup optimization: do a batch check per page -- `SELECT comment_github_id, github_updated_at FROM review_comments WHERE repo = $1 AND comment_github_id = ANY($2)` -- to avoid N individual getByGithubId calls. 6. Write comprehensive tests mocking store and octokit.
